# 小程序页面设计示意稿

本目录为 **微信开发者工具真机截图**，反映小程序当前实现状态。工程实现以 `miniprogram/` 下 WXML/WXSS 为准。

## 「今日任务」首页（`01-index-today.png`）主题约束

必须与 **`pages/index`** 一致：**室内植物养护**（浇水 / 施肥、植物昵称、今日养护），**禁止**画成习惯打卡、经验值、阅读运动复盘、一键领取、国风山水横幅等与绿植无关的界面。

## 示意稿约定（第二版已按此重生成）

1. **整屏竖图**：单台手机、**9:19 左右窄屏**，从顶栏到 Home 指示区尽量**完整一屏**，避免只保留上半截。
2. **问题诊断**：仅一台手机、**手机合理宽度**，禁止平板/超宽多栏布局。
3. **添加植物**：**拍照识别为主**——顶部大卡片/主按钮突出「拍照识别」，表单项次之。
4. **设置 · 天气与环境**：**`04-settings.png` 须与首页/植物页同一套视觉**（`#1c2e24` 导航、`#e8ece6` 底、白卡片弱阴影、深松绿主按钮、眉题/分区标题样式一致）；天气区可为 **小图标 + 极克制单色简图（曲线/温区）+ 文字**，禁止画成炫彩数据大屏、独立另一套 UI 语言。示意稿与实现均对齐 `pages/settings`。
5. **发现 · 知识库列表**：每条为 **左侧方图 + 右侧标题与描述** 的图文行卡片。
6. **养护知识详情**：**顶图 + 标题 + 正文**，正文中可穿插 **配图与说明**，杂志式排版；**`06-discover-detail.png` 必须为窄屏手机单列宽度**（与微信小程序内容区一致），禁止平板/超宽/多栏整页铺图。
